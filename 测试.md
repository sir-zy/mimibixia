明白了，你真正想要的是**动 HTML 结构本身**，把现有的左右分栏、导航、搜索、统计全部砍掉，改成一个**极简的记录本风格**——就像一本打开的笔记本，一页一页往下写东西。

下面我直接给你 **完整的改造方案**，包括要改哪些文件、改成什么样的 HTML，以及配套的 SCSS 让它真的像一本记录本。

---

## 📒 记录本风格的核心设计

- **结构**：单栏，无侧边栏，无复杂导航，无搜索框，无访问统计。  
- **内容**：首页就是一列日记/笔记条目，按时间倒序，每个条目只显示日期、标题（可选）、正文。  
- **视觉**：纸张质感背景（浅米色）、横线、手写或打字机字体、极简排版。  
- **页面**：只保留首页列表和单篇文章页（可选），去除归档、标签、关于等页面，或简化为顶部一个「关于」链接。

---

## 🔧 第一步：备份并修改文件

你需要修改/创建这几个文件（在 VSCode 里操作）：

### 1. `_layouts/default.html`（核心布局，改成单栏记录本）

替换为：

```html
<!DOCTYPE html>
<html lang="{{ site.lang | default: "zh-CN" }}">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{% if page.title %}{{ page.title }} | {% endif %}{{ site.title }}</title>
  <link rel="stylesheet" href="{{ "/assets/css/style.css" | relative_url }}">
  <!-- 引入打字机风格字体：Special Elite 或 手写体 -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Special+Elite&family=Courier+Prime&display=swap" rel="stylesheet">
</head>
<body>
  <div class="notebook">
    <header class="notebook-header">
      <h1 class="site-title">{{ site.title }}</h1>
      <p class="site-desc">{{ site.description }}</p>
      <!-- 极简导航：只保留首页和关于（可选） -->
      <nav class="simple-nav">
        <a href="{{ "/" | relative_url }}">📒 所有笔记</a>
        {% if site.pages contains "about" %}
        <a href="{{ "/about/" | relative_url }}">👤 关于</a>
        {% endif %}
      </nav>
    </header>

    <main class="notebook-content">
      {{ content }}
    </main>

    <footer class="notebook-footer">
      <p>📓 {{ site.time | date: '%Y' }} {{ site.title }} · 记录生活</p>
    </footer>
  </div>
</body>
</html>
```

### 2. `_layouts/home.html`（或者直接在 `index.html` 里循环）

如果你的首页以前是用 `index.html` 配合 `_layouts/default.html`，你需要确保 `index.html` 使用上面定义的布局，并循环输出文章。  
创建或修改 `_layouts/home.html`（如果没有就新建，然后用 `index.html` 的 `layout: home`）：

```html
---
layout: default
---

<div class="entries">
  {% for post in paginator.posts %}
  <article class="entry">
    <div class="entry-date">{{ post.date | date: "%Y-%m-%d" }}</div>
    {% if post.title %}
      <h2 class="entry-title"><a href="{{ post.url | relative_url }}">{{ post.title }}</a></h2>
    {% endif %}
    <div class="entry-content">
      {{ post.excerpt | strip_html | truncatewords: 50 }}
      {% if post.content contains '<!--more-->' %}
        <!-- 如果有 more 标签则显示链接，否则直接展开全文 -->
      {% endif %}
    </div>
  </article>
  {% endfor %}

  <!-- 分页：简单的上一篇/下一篇 -->
  <div class="pagination-simple">
    {% if paginator.previous_page %}
      <a href="{{ paginator.previous_page_path | relative_url }}">← 上一页</a>
    {% endif %}
    {% if paginator.next_page %}
      <a href="{{ paginator.next_page_path | relative_url }}">下一页 →</a>
    {% endif %}
  </div>
</div>
```

如果不想用分页，也可以直接用 `{% for post in site.posts %}` 全部列出，不设分页。

### 3. `_layouts/post.html`（单篇文章页，极简笔记详情）

```html
---
layout: default
---
<article class="post-full">
  <div class="post-date">{{ page.date | date: "%Y-%m-%d" }}</div>
  <h1 class="post-title">{{ page.title }}</h1>
  <div class="post-body">
    {{ content }}
  </div>
  <!-- 如果需要评论，可以保留 gitalk，也可以去掉 -->
</article>
<div class="back-link">
  <a href="{{ "/" | relative_url }}">← 返回笔记列表</a>
</div>
```

### 4. 删除或简化其他页面

- 可以保留 `about.md`，它也会使用 `default` 布局，形成一篇文章。
- 删除 `archive/index.html`、`tags/index.html` 等，或者将其内容改为简单的列表，并同样使用 `default` 布局。

## 🎨 第二步：SCSS 变成“笔记本”风格

修改 `_sass/style.scss`（或者你的主样式文件），添加或替换成以下内容：

```scss
// ========== 笔记本基础变量 ==========
$bg-notebook: #fdfaf3;        // 米黄纸张色
$color-line: rgba(0, 0, 0, 0.08); // 横线颜色
$font-text: 'Special Elite', 'Courier Prime', 'Courier New', monospace;
$font-heading: 'Special Elite', 'Courier Prime', monospace;
$text-color: #2c2c2c;

* {
  box-sizing: border-box;
}

body {
  margin: 0;
  background: #e8e4d9;        // 桌面/木纹背景，衬托本子
  font-family: $font-text;
  color: $text-color;
  line-height: 2;             // 与横线匹配
  min-height: 100vh;
  display: flex;
  justify-content: center;

  max-width: 700px;

  background-size: 100% 2rem; // 每行高度2rem（与line-height匹配）

// 头部
.notebook-header {
  text-align: center;
  margin-bottom: 3rem;
  border-bottom: 2px solid #ccc5b5;
  padding-bottom: 1.5rem;

    margin: 0;
    font-weight: normal;

  }
  .site-desc {

  }

}

// 条目列表

      margin-bottom: 0.2rem;
    }

      color: $text-color;

    }

      // 保留横线背景穿透
    }
  }
}

// 单篇文章内页

    color: #9b907b;
    margin-bottom: 1rem;
  }
  .post-title {
    font-size: 2rem;

  }
  .post-body {
    margin-top: 2rem;

      max-width: 100%;

      border-radius: 2px;
    }

  }

.back-link {
  margin-top: 3rem;
  text-align: center;

    text-decoration: none;

  }

// 分页

  margin-top: 3rem;

    margin: 0 1rem;

// 响应式：移动端缩小内边距

  }

```

如果你的 Jekyll 主题原本有 `_sass/` 下的其他文件，建议先清空或重命名它们，避免旧样式冲突。

## 🗂️ 第三步：在 VSCode 里执行操作

1. **备份**：把你的 `_layouts/`、`_includes/`、`_sass/` 先复制一份。
2. **替换布局**：按上面给出的代码，修改 `_layouts/default.html`，新建或修改 `_layouts/home.html`、`_layouts/post.html`。
3. **清理无用文件**：删除 `_includes/` 里的 `nav.html`、`footer.html`、`toc.html`、`analytics.html`、`gitalk.html`、`disqus.html`、`svg-icons.html`（保留 `head.html`，但可以把其中引入的 Google 字体替换成我给出的链接，或者简化为直接在 default.html 里引入）。
4. **样式重写**：把 `_sass/style.scss` 内容替换为上面的 SCSS，确保主样式文件（通常是 `assets/css/style.scss`）仍然 `@import` 它们。
5. **调整 `_config.yml`**（如果需要）：确保 `paginate: 10` 或你想要的数量，或者关闭分页直接用 `site.posts` 列出所有。
6. **本地测试**：运行 `bundle exec jekyll serve`，在浏览器查看效果。

如果你需要进一步调整（比如去掉日期只留正文、完全变成无标题的纯日记流），我也可以再给你微调方案。
