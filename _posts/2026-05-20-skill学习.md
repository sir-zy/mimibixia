Claude Code + DeepSeek V4 真的完全是贫民的最优配置了。

### 🤖 Skill 的认知转变

今天学习了 Claude Code + DeepSeek 的配置，我发现过去在使用中忽视了一个很重要的问题——我只是单纯地使用 API 的大模型工具，但对于更细节的东西，比如 Skill 的配置与使用都没有过多关注。或许是因为之前总是使用网页版的 AI 聊天，让我对 AI 形成了一种先入为主的思想，觉得 Skill 的配置与使用不是那么重要。

即使我已经开始使用一些厉害的 IDE，比如 DeepSeek TUI、Cursor、VS Code、Trae 等好用的工具，我仍然没有激发自己对 Skill 的思考。

我最开始对 Skill 产生想法是通过 Trae 的一次使用。在一次处理 PDF 的过程中，我发现 Trae 并不能完成对 PDF 的处理，这其实让我很震惊。因为对于我之前的认知来说，Trae 应该是可以处理诸如文档等简单任务的。又比如 Trae 对 node.js 命令的无法使用，这也让我很困惑。诸如此类的事情还发生在我使用小龙虾的时候。

特别是小龙虾，我通过向另一位老师询问得知，小龙虾会先定义工作空间，这样你的工作环境就会保持纯净。这也是我最开始的思考——或许这些 AI IDE 只会在需要的时候工作？但是后来我发现这就是我自己的问题。直到今天学习了 Claude Code + DeepSeek 的配置，我才对 Skill 进行了一定的学习。

Skill 的使用非常强大，不限于接入各种工具对文档进行处理，接入各种智能功能让我的 AI 能实现迭代思考，AI 可以变得更聪明。在之前这些功能被我限制在了小龙虾提供的一些工具上面，比如迭代工具和对文档的处理工具，现在我才发现，我以前只是先入为主地认为 Skill 只能被别人提供，而我并没有去深入学习这个东西。

虽然进行了一定的了解，但这远远不够。之后的方向上我应该去学习更多的 Skill 给我的 AI 使用，让我的 AI 变得更聪明，比如对文档的处理、对命令的使用等。

### 💻 Linux 命令学习

第二件事就是 Linux 命令的使用，目前我已经开始学习了很简单的命令，比如升级软件、安装新的软件等。

在 Linux 环境下，官方甚至将基础功能都不提供，你需要的时候再去获取。这也是我之前发现的——我安装了基础版的 Linux 系统，但是缺少的功能和工具以及对后续软件的适配都需要自己动手，这确实提升了我的一部分能力，比如对命令的了解以及对 Linux 的简单配置。

希望后续学习 Linux 不会太困难，至少能正常安装开发环境、安装一些软件什么的。未来我的开发模式就是 Windows 专门用来处理日常工作，而 Linux 专门用来开发和写代码。预计转折点会在明年上半年全面完成。
